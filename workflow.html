LAYER 0 Session & Security Gate
core/session_gate.py
Rate limiting, prompt-injection detection, chat history resolution ("it" → last entity), clarification state loading. If blocked → return immediately.
INPUT
query, session_id, user_id, conversation_history
OUTPUT
SessionGateResult: proceed, resolved_query, prior_clarifications
KEY OPERATIONS
Auth, rate limits, pronoun resolution, clarification state restore, injection check
EARLY EXIT
BLOCKED → return immediately, no pipeline
LATENCY TARGET
< 50ms (programmatic, no LLM)
KEY FUNCTIONS
trim_conversation_history_for_llm(), assert_required_clarifications_resolved(), get_session_gate()
LAYER 1 Regulatory Hard Gate
core/regulatory_gate.py core/language_sanitizer.py
Risk scoring (1–5), jurisdiction detection, BLOCK / REFRAME / PROCEED verdict. BLOCK returns educational response. REFRAME rewrites query with language constraints and continues pipeline.
INPUT
effective_query, session_context
OUTPUT
RegulatoryGateResult: verdict, risk_score (1–5), reframed_query, disclaimers, jurisdiction
VERDICTS
BLOCK REFRAME PROCEED
RISK SCORING
Level 1 (Safe/Educational) → Level 5 (CRITICAL: personalized invest advice). Dual: regex + LLM classification
REFRAME BEHAVIOR
safe_query = reframed_query; pipeline continues with language_constraints applied at L10
BLOCK BEHAVIOR
Returns educational message immediately. L2–L10 never run.
Language sanitization rules applied to ALL final output: prohibited phrases replaced (e.g. "you should buy" → "analysis indicates").
ROUTER Decision Router
core/decision_router.py
LLM-driven intent routing. Determines GREETING / TOP_NEWS / CURRENT_PRICE / FINANCIAL_EDUCATION / NON_FINANCIAL / NEEDS_CLARIFICATION / READY. Non-READY routes return early without entering the full pipeline.
ROUTE: GREETING / TOP_NEWS
Returns welcome + trending queries. Skips L2–L10 entirely.
ROUTE: NON_FINANCIAL
Polite redirect with financial reframe. Skips L2–L10.
ROUTE: NEEDS_CLARIFICATION
Return clarification questions. Client answers; next request with same session_id continues.
ROUTE: READY
READY → Continue to L3 Market Context and beyond.
KEY FUNCTIONS
_classify_intent_llm(), _get_trending_queries_async(), _get_trending_news_async(), _detect_confirmation_and_entity()
TRENDING QUERIES
Provided by core/trending_queries_manager.py for greeting/market-pulse responses
LAYER 3 (PRE-L2) Market Context Builder
core/market_context.py
Builds immutable MarketContext: US/India/EU market hours status, earnings season flag, major scheduled events (FOMC, CPI), circuit breakers, region, currency. Injected into all downstream layers (L6.5, L7, L10).
OUTPUT FIELDS
us_market_status, india_market_status, eu_market_status, is_earnings_season, major_events_today, detected_region, detected_currency
IMMUTABILITY
MarketContext is frozen once built. Consistent across all downstream components in the same request.
INJECTED INTO
L6.5 frame_scenarios(), L7 enriched_context, L10 response composition
LAYER 2 Query Classification
core/query_classifier.py
Classifies query type (SCREENING / SINGLE_STOCK / RESEARCH / MACRO / EDUCATIONAL / COMPOUND), subtype, complexity (SIMPLE → VERY_COMPLEX), entities, timeframe, and ambiguity. Mandatory ambiguity → clarification response. Optional ambiguity → proceed with stated assumptions.
INPUT
safe_query, conversation_history, prior_clarifications
OUTPUT
ClassifiedQuery: query_type, entities, complexity, subtype, ambiguity, agent/engine hints
QUERY TYPES
SCREENING (technical/fundamental/multi-factor/short-squeeze), SINGLE_STOCK, RESEARCH, MARKET_INTELLIGENCE, EDUCATIONAL, COMPOUND
COMPLEXITY
SIMPLE (1–2s) | MODERATE (5–15s) | COMPLEX (15–30s) | VERY_COMPLEX (30–60s)
POST-CLASSIFICATION
Pipeline cache check, survivorship bias filter applied to entities, clarification guard (assert before L4)
COMPOUND HANDLING
Decomposes into sub-queries → pipeline run per sub-query → results merged
LAYER 3A Entity Resolution
core/layer3_entity_resolver.py core/entity_resolver.py
Resolves human-readable names to canonical tickers. "Apple" → AAPL, "Reliance" → RELIANCE.NS. Single source of truth for all downstream API calls and agents. Primary: yFinance Lookup. Fallback: OpenFIGI Search API. LLM + Web Search enhanced.
RESOLUTION CHAIN
1. LLM entity extraction → 2. yFinance lookup → 3. OpenFIGI fallback → 4. Web search discovery
OUTPUT
Resolved tickers enriched into classification entities. Used by L4 and all CRA agents.
EXCHANGE HANDLING
RELIANCE.NS (NSE) vs RELIANCE.BO (BSE); US/India/AU/EU disambiguation
LAYER 4 Data Acquisition & Validation
core/api_orchestrator.py core/data_validator.py
Parallel fetch of market data, news, macro, crypto, filings from 21 external API providers. Per-provider rate limiting via token buckets. Data validated with quality scoring (0.0–1.0). Quality gate: score <0.4 = REJECTED.
ROUTING
Routes by DataType (CRYPTO, US_STOCK, MACRO_US, INDIA_STOCK) and QueryType with fallbacks
RATE LIMITING
TokenBucketRateLimiter per provider, ProviderRateLimiterRegistry, exponential backoff
QUALITY GATES
<0.40 REJECT | 0.40–0.70 WARN | 0.70–0.90 ACCEPTABLE | >0.90 HIGH QUALITY
VALIDATION CHECKS
Price anomalies, OHLC logic, missing data gaps, restatements, negative market cap, outliers (3σ)
KEY CLASSES
IntelligentAPIOrchestrator, ParallelExecutionEngine, LLMEndpointSelector, MacroDataCache
UPSTREAM DATA REUSE
L4 payload passed to CRA as upstream_data to avoid duplicate fetches at Layer 7
LAYER 5 Retrieval Confidence
core/retrieval_confidence.py
Per-source confidence scoring: source agreement (do 3 sources agree?), freshness weighting (real-time → stale), outlier flagging (>3σ). Composite score attached to data_result and injected into L7 enriched_context and L10 metadata.
SOURCE AGREEMENT
3 sources agree → HIGH | 2 agree → MEDIUM | all differ → LOW
FRESHNESS WEIGHTS
Real-time: 1.0 | Today: 0.9 | Yesterday: 0.7 | Last week: 0.5 | >2 weeks: 0.3
OUTPUT
RetrievalConfidenceResult with tags (ConfidenceTag), stored in data_result["retrieval_confidence"]
LAYER 6 Quantitative Computation
core/computation_engine.py
⚠ ABSOLUTELY NO LLMs. Pure deterministic math (NumPy/Pandas/SciPy). Technical indicators, anomaly detection, regime classification, screening, statistical analysis. All numeric outputs originate here.
ANOMALY DETECTION
Price/volume spikes, zero volume, OHLC violations, large gaps, stale data, probable splits. Runs first.
TECHNICAL ANALYSIS
RSI, MACD, Bollinger Bands, ATR, OBV, pivots, SMA/EMA, golden/death cross, composite signal
COMPOSITE SCORING
Momentum×0.30 + Trend×0.25 + Volume×0.20 + Volatility×0.15 + Pattern×0.10
STATISTICAL ENGINE
Regression, correlation, Granger causality, VAR, event studies (MacKinlay 1997)
REGIME CLASSIFICATION
expansion / contraction / transition / high_volatility from price and momentum signals
KEY DESIGN
Fully vectorized operations across DataFrames. Never loop over individual stocks.

core/contradiction_engine.py

METHOD
frame_scenarios(computation_result, regime, market_context) → ScenarioFrame
OUTPUT
Bull/Base/Bear scenarios with prior probabilities that sum to 1.0
USED BY

Layer 7 Deep Dive

CRA — Chief Research Agent Workflow

CRA is a LangGraph StateGraph (v7.0). It orchestrates all 17 specialist agents dynamically via task planning, then synthesizes their outputs into one institutional-grade executive summary.

Design principle: CRA uses task-planning LLM to decompose the query into analytical tasks and assign each to one agent with a specific directive. No static agent lists — selection is fully dynamic per query. All selected agents run in parallel via asyncio.gather().
01
Cache Check
If identical query is cached for this session, return immediately. Skip all subsequent nodes.
02
Parse Intent
LLM extracts: entities, entity types, intent, time_horizon, require_rag flag, thematic focus.
03
Task Planning
LLM decomposes query into tasks. Maps each task to one agent. Writes per-agent directive (e.g. "Assess RSI and MACD levels for AAPL"). Outputs: selected_agents, agent_directives, task_plan.
04
Mandatory API Fetch
Async fetch of price/news/macro for resolved entities. Reuses upstream_data from L4 when available to avoid duplicate calls.
05
Parallel Data Fetch
Web search + RAG (for MEDIUM/BIG queries) run in parallel. RAG: multi-query, HyDE, reranking via Pinecone. Results go into shared context for all agents.
06
Parallel Agent Dispatch
Each selected agent receives (query_id, entity, context + analytical_directive). All run concurrently. Each returns: status, confidence, findings, signal, key_metrics.
07
Validate Outputs
Verify each agent addressed its assigned task. Optional one recovery re-run for critical task failures. Capped to prevent latency blowup.
08
Aggregate Results
Merge all agent findings into aggregated_findings. Compute weighted confidence and agent_agreement score. Build aggregated_findings text for synthesis input.
09
LHIS Planning
Optional multi-horizon planning: Tactical (1-day) / Operational (1-month) / Strategic (1-year). Skipped for SMALL queries.
10
Generate Output
LLM synthesis: aggregated_findings → single institutional executive summary. Handles agreement (combine) vs disagreement (explore) vs scenarios. Generates follow_up_suggestions.
11
Quality Validation
GPT-4o validates numerical accuracy, tone, data usage, and that output is grounded in actual computation (not hallucinated). Used for MEDIUM/BIG queries.
12
Compliance Validation
Enforces jurisdictional disclaimers. Checks prohibited phrases. Applies language_constraints from L1 Regulatory Gate. Then returns to pipeline.
Universal Agent Contract: Every agent receives (query_id, entity, context) where context includes analytical_directive, price_data, news_data, macro_data, rag_context, computation_results, time_horizon, intent, current_date, and data_contract rules (no fabrication, cite data). Every agent returns {status, confidence, findings, signal, entity, key_metrics, ...}. CRA uses findings for aggregation and signal + confidence for agreement scoring.

Layer 6 · No LLMs

8 Computation Engines

All numeric outputs originate here. Pure NumPy/Pandas/SciPy/StatsModels. LLMs never touch these numbers — they only explain results from these engines.

Anomaly Detection Engine
scipy, scikit-learn (isolation forest)
Price/volume spikes, zero volume on trading days, OHLC violations, large price gaps, stale data detection, probable stock splits. Runs first before all other engines.
Technical Screening Engine
pandas, numpy, ta-lib
Vectorized computation of RSI 14, MACD 12/26/9, Stochastic 14/3/3, SMA 50/200, EMA 9/21/50, ADX, Bollinger %B, OBV, A/D Line across entire stock universe. <8s for 450 stocks × 15 indicators.
Fundamental Scoring Engine
pandas, numpy
P/E, P/B, ROE, revenue growth, margin compression, earnings quality scoring, restatement detection, industry median benchmarking, outlier detection.
Composite Screening Engine
pandas, numpy, scikit-learn
Multi-factor composite scoring: Valuation×0.25 + Catalyst×0.25 + Sentiment×0.25 + Macro×0.25. Normalization, z-score weighting, signal agreement percentage.
Short Squeeze Detection Engine
pandas, numpy, scipy
Squeeze_Score = SI×0.25 + Days-to-Cover×0.20 + Momentum×0.20 + Volume×0.15 + Options×0.10 + Borrow×0.07 + Technical×0.03. Multi-source data fusion.
Statistical Research Engine
statsmodels, scipy, numpy
Granger causality (F-tests, p<0.05, lag 1–10), VAR models with impulse response, Pearson/Spearman correlation, rolling correlations (90-day), event studies (MacKinlay 1997), variance decomposition.
Macro Sensitivity Engine
statsmodels (OLS regression)
Rate beta computation, stock returns vs macro variable changes, sensitivity scores, regime-conditional regression, interest rate impact quantification.
Regime Classification Engine
numpy, scipy
Classifies market periods: expansion / contraction / transition / high_volatility from price momentum and macro indicators. Used by L6.5 scenario framing and L8 contradiction engine.

Advanced Systems

Supporting Intelligence Systems

Four advanced systems run alongside the main pipeline: QPU (cost tracking), RAG (knowledge retrieval), RL (agent weighting), MoE (hyperparameter optimization), and LHIS (multi-horizon planning).

⚡
QPU
Query Processing Unit · core/qpu/
Normalized cost unit: 1 QPU ≈ $0.0025 USD. Tracks tokens, API weight, model size, and vector retrieval operations per query. Enforces tier-based daily limits.
  • calculator.py — QPU = f(tokens, API calls, model, retrieval). Cost = QPU × $0.0025
  • balance_manager.py — Per-user, per-tier balances; atomic deduction via PostgreSQL
  • middleware.py — Pre-call balance check; post-call QPU deduction
  • transaction_logger.py — Full audit trail, cache hit rates, analytics
  • nightly_allocation_job.py — Daily resets via scheduler (Free / Essentials / Professional / Institutional tiers)
  • API: GET /api/qpu/balance/{user_id}, /transactions, /dashboard
📚
RAG v2.0
Retrieval Augmented Generation · rag/
Dynamic retrieval of academic/professional methodologies from ~300-book Pinecone vector store. Augments CRA and agent context with grounded financial research.
  • Adaptive router: FAST / STANDARD / DEEP by query complexity
  • Multi-query: LLM generates multiple query variants for broader retrieval
  • HyDE: Hypothetical document embedding for similarity search (skipped in FAST)
  • Metadata filtering: asset_class, entities, benchmarks, agent_access
  • Cross-encoder reranking: LLM scores candidates 0–10; re-sort
  • Iterative confidence: DEEP mode re-retrieves if quality < 7/10 (max 2 iterations)
  • Parent document expansion: Expand retrieved chunks to neighbours
  • Ingestion: PDFs → FinanceAwareChunker → text-embedding-3-large → Pinecone
🧬
RL
Reinforcement Learning · rl/
Trains policies for agent selection, weighting, and multi-horizon behavior. Collects feedback for continuous online learning. Integrated with MoE and LHIS.
  • FinancialAgentEnvironment: 7 agents, 70-dim state space
  • HierarchicalEnvironment: Multi-horizon LHIS-compatible
  • RLTrainer: PPO / A2C / SAC / TD3 via Stable Baselines3; MLflow tracking

Design Boundaries: Numbers ONLY from computation_engine + API data — LLMs only for classification, synthesis, RAG, task planning, reranking. BLOCK at L1 → immediate return. REFRAME → pipeline continues. Mandatory clarification blocks pipeline at L4 guard (assert_required_clarifications_resolved). Optional ambiguity → proceed with stated defaults. Per-ticker quality gate <0.4 → rejected before computation. Confidence propagated: data → L5 → L7 enriched_context → L9 decomposition → L10 metadata. Pipeline owns L0–L6 and L8–L10. CRA owns ONLY L7. asyncio.Lock(_stats_lock) protects PipelineOrchestrator._stats mutations.
